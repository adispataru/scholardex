# H23.5 Route Map and Closeout

Date: 2026-03-13  
Status: H23 closeout route map for canonical UI ownership, retained compatibility shims, and verification expectations.

> Superseded for shared MVC routing by H25.2 (2026-03-13): legacy public alias families (`/rankings/*`, `/scholardex/forums*`, `/core*`, `/urap*`) and admin duplicate read GET aliases under `/admin/rankings/*` were removed. Use `docs/h25.1-canonical-route-ownership-contract.md` + `TASKS.md` H25 notes as the current route ownership source.
>
> Superseded for user-owned MVC routing by H25.3 (2026-03-13): legacy user aliases (`/user/activityInstances*`, `/user/individualReports*`, `/user/publications/scopus_tasks`, `/user/tasks/scopus/updateCitations`, `/user/publications/exportCNFIS2025`, `/user/export/cnfis`, `/user/rankings/{id}`) were removed. Canonical user routes are kebab-case under `/user/*`.

## 1. Summary

H23 consolidated the UI around Scholardex-first forum navigation and category-first ranking navigation.

The steady-state distinction after H23 is:

- canonical MVC/UI routes are Scholardex/category-first,
- legacy MVC Scopus/WoS paths remain only as compatibility redirects,
- legacy API namespaces remain intentionally stable,
- H23 introduced one new public API surface for the paged WoS category UI.

This document is the operational route map for post-H23 UI ownership. Use `docs/h23.1-transitional-debt-inventory.md` for debt classification, not for the current canonical route map.

## 2. Canonical MVC Route Families

### 2.1 Public canonical routes

- `/scholardex/forums`
  - canonical public forum discovery page
  - includes all known forums
  - absorbs former WoS-listing behavior via WoS status/filtering
- `/scholardex/forums/{id}`
  - canonical forum detail page
  - journal forums show WoS detail or a deterministic “not indexed by WoS” state
  - conference proceedings show the CORE placeholder
  - books/book series show the book-ranking placeholder
- `/rankings/categories`
  - canonical WoS category listing page
  - paged via `/api/rankings/categories`
- `/rankings/categories/{key}`
  - canonical WoS category detail page
  - links journals back to `/scholardex/forums/{id}`

### 2.2 Admin canonical routes

Canonical admin catalog pages live under `/admin/scholardex/**`, especially:

- `/admin/scholardex/forums`
- `/admin/scholardex/forums/edit/{id}`
- `/admin/scholardex/authors`
- `/admin/scholardex/authors/edit/{id}`
- `/admin/scholardex/affiliations`
- `/admin/scholardex/affiliations/edit/{id}`
- `/admin/scholardex/publications`
- `/admin/scholardex/publications/search`
- `/admin/scholardex/publications/citations`

Admin forum discovery is view-first:

- forum rows link to `/scholardex/forums/{id}`,
- edit remains an explicit admin action, not the primary row target.

## 3. Retained Compatibility MVC Redirects

The following routes are intentionally retained as compatibility shims and are not canonical UI ownership points:

- `/rankings/wos`
  - redirects to `/scholardex/forums?wos=indexed`
- `/rankings/wos/{id}`
  - redirects to `/scholardex/forums/{id}`
- `/admin/scopus/**`
  - redirects to `/admin/scholardex/**`
- `/admin/scopus/venues*`
  - redirects to `/admin/scholardex/forums*`

These routes remain documented because H23 did not remove all compatibility shims. They should not be treated as first-class navigation targets in templates, scripts, or future route docs.

## 4. API Route Ownership After H23

### 4.1 Intentionally stable legacy APIs

These remain stable and intentionally named as-is:

- `/api/scopus/**`
- `/api/rankings/wos`

H23 did not rename these APIs, even though the canonical MVC/UI moved to Scholardex/category-first navigation.

### 4.2 New H23 API surface

H23 added:

- `/api/rankings/categories`

Purpose:

- provide paged category list data for `/rankings/categories`
- preserve the same page envelope style used by other ranking-list APIs

H23 also introduced dedicated page/list DTOs for this paged WoS category UI. This is the only new API/DTO surface added in H23; broader API renaming remains deferred.

## 5. Verification Contract

The H23 verification entrypoint is:

```bash
npm run verify-h23-ui
```

Use it when touching:

- canonical Scholardex forum pages,
- paged WoS category pages,
- admin Scholardex catalog navigation,
- route-level UI links/assets/redirect expectations introduced by H23.

This command is contributor-facing documentation, not a CI-required gate by default.

## 6. H23 Closeout Notes

H23 outcome:

- canonical public UI navigation is now Scholardex/category-first,
- WoS rankings no longer own the primary forum listing or detail flow,
- category pages now exist as real paged public pages,
- compatibility redirects remain explicit and documented,
- stable API naming was intentionally preserved for `/api/scopus/**` and `/api/rankings/wos`.

Follow-on work after H23 should use this route map as the source of truth for UI ownership and use `docs/h23.1-transitional-debt-inventory.md` as the source of truth for remaining transitional debt.
