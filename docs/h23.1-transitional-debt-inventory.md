# H23.1 Transitional Debt Inventory and Classification

Date: 2026-03-13

Scope: internal legacy/transitional flow retirement and simplification for H23.

## Classification Rules

- `remove now`: dead or misleading debt not required for runtime, parity gates, or public contract preservation.
- `keep intentionally`: still required for an explicit contract or operational tool.
- `defer`: real debt, but removal belongs to H23.2, H23.3, or a later dedicated slice because of public contract preservation or higher-risk wiring.

Owner defaults used in this inventory:

- Platform Backend Owner: runtime routing, redirects, internal naming debt, task follow-through.
- Frontend Owner: template/script allowances and frontend guardrails.
- Reporting Domain Owner: reporting lookup debt affecting CNFIS/WoS lookup flows.

## 5. Inventory Conclusions for H23 Follow-On Work

- Status update (2026-03-13): H23.2-H23.4 have now landed. Canonical UI routing is described in `docs/h23.5-route-map-and-closeout.md`; this inventory remains the source of truth for debt classification and deferred cleanup ownership.
- No ambiguous hotspots remain in the H23.1 scope. Every identified item is classified as `keep intentionally` or `defer`; no `remove now` candidates were confirmed from current runtime evidence.
- Dual-read Mongo adapters are not dead code today. They are operational H22.6 tooling and should stay isolated from runtime cleanup until the dual-read gate itself is retired.
- Public Scopus-branded routes and DTOs are active contract debt, not accidental leftovers. H23.1 should inventory them but not rename them.
- The biggest immediate cleanup opportunity for H23.2/H23.3 is internal naming and wiring simplification:
  - remove “switchable” semantics from Postgres-only runtime facades,
  - normalize admin route/template compatibility naming,
  - continue shrinking frontend inline-script and legacy-script allowances.
- `TASKS.md` does not require wording changes based on the current inventory. The task statement already matches the hotspots found in the repo, so this document can serve as the H23.1 source of truth.
