# H14.1 Canonical WoS Data Contracts

## Summary
This document locks the schema and source-policy contract for H14 Approach 3 before identity/parser/migration work starts.

Operational policy:
- AIS/RIS canonical operational authority: government files (`GOV_AIS_RIS`).
- Official WoS extracted JSON (`OFFICIAL_WOS_EXTRACT`) is authoritative supplemental data and the allowed source for IF facts.
- Edition is first-class and normalized (`SCIE|SSCI|AHCI|ESCI|OTHER|UNKNOWN`), not embedded in category strings.
- H19.1 reference: authoritative multi-source Scholardex identity contract is locked in `docs/h19.1-multisource-identity-contract.md`.
- H19.2 reference: forum/journal cross-source linking policy is defined in `docs/h19.2-forum-keying-merge-contract.md`.

## Collections and Views

### `wos.import_events`
- Purpose: immutable ingestion ledger for replay and lineage.
- Fields:
  - `id`
  - `sourceType` (`GOV_AIS_RIS|OFFICIAL_WOS_EXTRACT`)
  - `sourceFile`
  - `sourceVersion`
  - `checksum`
  - `payloadFormat` (for example `excel-row`, `json-item`)
  - `payload` (raw serialized payload snippet/item)
  - `sourceRowItem` (row number / item index)
  - `ingestedAt`

### `wos.journal_identity`
- Purpose: canonical journal identity and aliasing.
- Fields:
  - `id` (`journalId`)
  - `primaryIssn`
  - `eIssn`
  - `aliasIssns`
  - `title`
  - `normalizedTitle`
  - `active`
  - `createdAt`
  - `updatedAt`

### `wos.metric_facts`
- Purpose: metric value facts by journal/year/metric.
- Canonical uniqueness rule:
  - `(journalId, year, metricType, editionNormalized)` where `editionNormalized` may be `UNKNOWN` if source does not expose edition at metric level.
- Fields:
  - `id`
  - `journalId`
  - `year`
  - `metricType` (`AIS|RIS|IF`)
  - `value` (`-999` normalized to missing/null before persistence)
  - `editionRaw`
  - `editionNormalized`
  - `sourceType`
  - `sourceEventId`
  - `sourceFile`
  - `sourceVersion`
  - `sourceRowItem`
  - `createdAt`

### `wos.category_facts`
- Purpose: category+edition level quarter/rank facts.

- Fields:

## Edition Normalization Rules
- `SSCI` token => `SSCI`.
- `SCIE` token => `SCIE`.
- `SCIENCE` token => `SCIE`.
- no recognized token:
  - blank/null => `UNKNOWN`
  - non-blank => `OTHER`.
- Bundled source value containing both SCIE and SSCI:
  - emit two normalized records with the same lineage (`sourceEventId`, `sourceFile`, `sourceVersion`, `sourceRowItem`).

## Source Precedence Rules
- AIS/RIS conflicts on same canonical key:
  - canonical operational winner is `GOV_AIS_RIS`.
  - `OFFICIAL_WOS_EXTRACT` values are retained with lineage for audit/replay.
- IF facts:
  - accepted only from `OFFICIAL_WOS_EXTRACT`.
  - no operational dependency on current government JIF ingestion.
  - H14.6 enforcement: active government WoS ingestion paths accept AIS/RIS only; JIF gov ingestion is retired.
