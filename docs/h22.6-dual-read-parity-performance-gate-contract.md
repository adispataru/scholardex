# H22.6 Dual-Read Parity + Performance Gate Contract

## Scope
H22.6 provides a deterministic gate that executes first-wave H22.4 read scenarios against both providers:
- Mongo legacy read adapters.
- PostgreSQL read adapters.

Covered scenarios:
- `wos.issn-ranking.lookup`
- `wos.top-rankings.count`
- `scopus.author.search`
- `scopus.forum.search`
- `scopus.affiliation.search`
- `admin.scopus.publication.search`
- `admin.scopus.citations.view`

## Enforcement Rules
- Functional parity is strict after deterministic normalization:
  - hard-fail on missing/extra entities,
  - hard-fail on scalar/value drift,
  - hard-fail on runtime error/status mismatch.
- Performance gate is per scenario:
  - compute Mongo and Postgres p95 in the same gate run,
  - require `postgres_p95 <= threshold * mongo_p95`.
- Default threshold: `1.2` (`core.h22.dual-read-gate.p95-ratio-threshold`).

## Runtime Contract
Service contract:
- `DualReadGateService#runFullGate()`
- `DualReadGateService#latestStatus()`

Persisted artifacts (`reporting_read` schema):
- `dual_read_gate_run`
- `dual_read_gate_scenario_run`

Each run stores:
- run metadata (`run_id`, status, sample size, threshold, timings),
- scenario-level parity/performance flags,
- latency metrics (`avg`, `p95`, ratio),
- mismatch sample for triage.

## Admin Orchestration
Initialization admin endpoints:
- `POST /admin/initialization/postgres/dualReadGate/run`
- `POST /admin/initialization/postgres/dualReadGate/showStatus`
- `GET /admin/initialization/postgres/dualReadGate/status`

Initialization page includes explicit H22.6 gate actions.

## CI Contract
Dedicated test task:
- `./gradlew testH226DualReadGate`

Build must fail when any scenario fails parity or exceeds p95 ratio threshold.

## Configuration
Postgres profile properties:
- `core.h22.dual-read-gate.sample-size` (default `5`)
- `core.h22.dual-read-gate.p95-ratio-threshold` (default `1.2`)

## Non-Goals
- H22.6 does not change runtime read-store routing.
- H22.6 does not alter ingest/canonical write paths.
