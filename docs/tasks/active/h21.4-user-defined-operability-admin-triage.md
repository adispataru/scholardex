# H21.4 USER_DEFINED Operability and Admin Triage

## Scope

This document locks H21.4 behavior for USER_DEFINED operability and admin triage integration.

H21.4 delivers:
- a dedicated read-only admin triage page for USER_DEFINED diagnostics,
- dedicated USER_DEFINED initialization/maintenance endpoints under `/admin/initialization/user-defined/*`,
- source-accurate metrics/logging and source-filtered operability counters for USER_DEFINED triage.

## Locked Decisions

- Wizard route and UX remain unchanged at `/user/publications/add`.
- No new admin approval workflow is introduced in H21.4.
- New triage surface is diagnostics-only with deep links:
  - `/admin/source-links?source=USER_DEFINED`
  - `/admin/conflicts?incomingSource=USER_DEFINED`
  - `/admin/initialization` (USER_DEFINED section)
- Existing `/admin/initialization/scopus/*` behavior remains intact; no migration/removal in this slice.

## Admin Contracts

- New route: `GET /admin/user-defined-triage`.
- New maintenance endpoints:
  - `POST /admin/initialization/user-defined/buildFacts`
  - `POST /admin/initialization/user-defined/canonicalize`
  - `POST /admin/initialization/user-defined/runAll`
- Maintenance orchestration order:
  1. USER_DEFINED fact build (optional in canonicalize-only run)
  2. USER_DEFINED canonicalization
  3. optional source-link reconcile
  4. optional edge reconcile
  5. optional projection rebuild

## Operability Expectations

- Triage counts are source-filtered for USER_DEFINED:
  - source-link counts by `source + state`,
  - identity-conflict counts by `incomingSource + status`.
- Recent triage rows surface lineage envelope fields:
  - `sourceEventId`
  - `sourceBatchId`
  - `sourceCorrelationId`
- Canonical-build operability logs and metrics include explicit USER_DEFINED labeling in mixed-source processing so operators can distinguish USER_DEFINED workload from SCOPUS-only runs.

## Non-Goals

- No route/controller/template changes to user onboarding flow.
- No historical backfill/replay migration in this slice.
- No direct mutating actions from the triage page.
