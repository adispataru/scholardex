# H21.2 USER_DEFINED Wizard Submit Migration

Date: 2026-03-14  
Status: active implementation contract for H21.2.

## 1. Summary

This document locks H21.2 implementation behavior for migrating wizard submission into first-class canonical `USER_DEFINED` ingest.

Locked direction:

- keep route/UI flow unchanged at `/user/publications/add`,
- migrate wizard submit lineage source to `USER_DEFINED`,
- emit deterministic `USER_DEFINED` source-record IDs for new wizard-created forum/publication lineage,
- preserve existing selected forum IDs when they already exist in projections,
- do not include historical backfill/replay in H21.2.

## 2. Migration Decisions

- Ingest source token for wizard submission is `USER_DEFINED`.
- New forum IDs emitted by wizard contract helper use `USER_DEFINED:FORUM:*`.
- Publication source record IDs emitted by wizard contract helper use `USER_DEFINED:PUBLICATION:*`.
- Wizard EID emitted by submit flow uses `USER_DEFINED:EID:*` and is derived without hardcoded `MANUAL:` assumptions.
- Batch/correlation lineage remains deterministic and replay-safe.

## 3. Compatibility Locks

- Payload wire-shape remains stable for fact-builder compatibility (`source_id`, `author_ids`, `afid`, etc.).
- Existing wizard metadata keys remain present (`wizardSubmitter*`, `wizardSourceRecordId`, `wizardSubmittedAt`).
- Source-link alias normalization (`USER_PUBLICATION_WIZARD -> USER_DEFINED`) remains in place; alias removal is out of scope.

## 4. Scope And Non-Goals

- No new routes, controller signatures, or template flow changes.
- No standalone author/forum/affiliation onboarding expansion in this subtask.
- No one-time migration/backfill of historical `USER_PUBLICATION_WIZARD` / `MANUAL:*` records in H21.2.

## 5. Verification Gate

- `PublicationWizardFacade` tests assert `USER_DEFINED` ingest source and deterministic `USER_DEFINED` key prefixes.
- Tests assert selected existing forum IDs are preserved.
- Tests assert wizard submit no longer relies on `MANUAL:` substring assumptions for EID/batch derivation.
