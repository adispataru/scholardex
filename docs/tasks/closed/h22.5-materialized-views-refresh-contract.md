# H22.5 Materialized Views + Refresh Contract

Date: 2026-03-12
Status: implemented.

## Summary
H22.5 adds read-side materialized views for heavy SQL paths and a deterministic refresh strategy:
- projection-coupled refresh after successful changed slices,
- admin-triggered manual refresh operations,
- concurrent refresh mode (`REFRESH MATERIALIZED VIEW CONCURRENTLY`).

## Materialized Views
- `reporting_read.mv_wos_top_rankings_q1_ais`
  - pre-aggregated top-rankings counts by `(year, category_name_canonical, edition_normalized)`.
- `reporting_read.mv_scholardex_citation_context`
  - citation context rows keyed by `(cited_publication_id, citing_publication_id)` with citing publication fields used by admin/report citation views.

Unique indexes are present for concurrent refresh safety.

## Refresh Triggers
- Projection-coupled:
  - in `JdbcPostgresReportingProjectionService`, after each successful changed slice commit:
    - `wos` -> refresh WoS MV,
    - `scopus` -> refresh citation MV.
- Manual admin controls:
  - refresh all,
  - refresh single slice (`wos` or `scopus`),
  - show latest refresh status,
  - JSON status endpoint.

## Failure Contract
- MV refresh failures are fail-fast.
- For projection-coupled refresh:
  - slice/run is marked failed,
  - checkpoint is not advanced for failed slice.

## Runtime Read Contract
- `PostgresReportingLookupFacade#getTopRankings` reads from `mv_wos_top_rankings_q1_ais`.
- `PostgresScholardexAdminReadPort#buildPublicationCitationsView` reads citation chain rows from `mv_scholardex_citation_context`.
- Public endpoint payloads remain unchanged.
