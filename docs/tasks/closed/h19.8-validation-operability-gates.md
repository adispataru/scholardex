# H19.8 Validation, Parity, and Operability Gates
Status: active indexed source-of-truth document.

## Summary

This document defines the H19.8 gate contract for implemented sources in this slice:

- `SCOPUS`
- `WOS`
- current manual/user wizard path (`USER_DEFINED`)

No performance threshold is enforced in CI for H19.8. Gates are functional, deterministic, parity, and operability focused.

## Local and CI Commands

Run locally:

```bash
npm run verify-h19-baseline
npm run verify-h08-observability-guardrails
./gradlew help --stacktrace
```

CI enforcement:

- `.github/workflows/h09-quality-gates.yml` (`guardrails` job) runs `verify-h19-baseline`.
- Existing guardrails continue to run, including observability checks.

## Gate Failure Conditions

H19.8 fails when any of the following fails:

- Canonical replay/idempotence tests for publication/citation/source-link/edges.
- WoS onboarding deterministic linking tests.
- Wizard canonical ingest path tests and projection visibility tests.
- Generic conflict admin lifecycle tests (`OPEN -> RESOLVED|DISMISSED`).
- Source-link FSM and replay traceability tests.
- Cutover guardrails preventing source-silo runtime regressions.
- Observability marker checks for H19 metrics and triage log envelopes.

## Triage Workflow

When H19.8 gates fail, inspect in this order:

1. `/admin/conflicts` for open generic identity conflicts and recent reason codes.
2. `/admin/source-links` for `UNMATCHED|CONFLICT|SKIPPED` entries by entity/source.
3. Canonical build/reconcile logs with `H19_TRIAGE` markers:
   - `canonical_materialization`
   - `canonical_build`
   - `source_link_reconcile`
   - `edge_reconcile`
4. Actuator metrics:
   - `core.h19.canonical.build.*`
   - `core.h19.reconcile.*`
   - `core.h19.source_link.transitions`
   - `core.h19.identity_conflict.created`
   - `core.h19.identity_conflicts.open`
   - `core.h19.source_links.state`

## Replay and Recovery Sequence

Use this sequence for deterministic recovery:

1. Run canonical build for affected entity (or `all`) from initialization/admin ops.
2. Run source-link reconcile.
3. Run edge reconcile.
4. Rebuild projections.
5. Re-run `npm run verify-h19-baseline`.

Expected property: reruns are idempotent and must not create duplicate canonical facts, links, or edges.
