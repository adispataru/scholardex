# H22.4 Query-Layer Cutover Contract (Mongo -> Postgres Reads)
Status: active indexed source-of-truth document.

## Summary
H22.4 originally introduced runtime read-store switching for first-wave reporting/admin read surfaces.
Post H22.9, migrated H22 runtime reads are Postgres-first by default in active request paths.

Mongo remains authoritative for ingest/event/canonical writes.

## Cutover Scope
The following surfaces are migrated together and resolve to Postgres adapters in runtime:
- `ReportingLookupPort` (via `SwitchableReportingLookupFacade`)
- `ScholardexAuthorQueryService`
- `ScholardexForumQueryService`
- `ScholardexAffiliationQueryService`
- `ScholardexAdminReadFacade`

## Startup Guard (All-Or-Nothing)
`PostgresReadCutoverGuard` enforces readiness when Postgres profile/services are active:
- requires H22.4 Postgres adapters to be present in the application context
- requires H22.3 projection checkpoints for both slices:
  - `wos`
  - `scopus`

If readiness checks fail, startup aborts with explicit error.

## SQL Read Contract
Postgres adapters query only `reporting_read.*` objects from H22.2/H22.3.
No endpoint payload contract changes are introduced in H22.4.

## Rollback Contract
Rollback is operational (rebuild/recovery), not a normal runtime read-store toggle for migrated paths.
No data rollback is required because H22.4+ changes only read/query path authority.
