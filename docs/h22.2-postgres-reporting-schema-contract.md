# H22.2 Postgres Reporting Schema Contract

Date: 2026-03-11  
Status: active schema baseline for H22 read-core migration.

## Summary

This document locks the H22.2 PostgreSQL schema contract for first-wave migrated read surfaces:
- WoS ranking/scoring read core,
- Scholardex publication/author/forum/affiliation/citation read core,
- edge/fact traversal paths used by reporting/admin reads.

This slice delivers schema and migration infrastructure only. Runtime query cutover remains in H22.4.

## Decision Locks

- Scope: WoS + Scopus/Scholardex read core for first-wave reporting/admin reads.
- Migration mechanism: Flyway versioned SQL migrations.
- Namespace: `reporting_read`.
- Enum strategy: native Postgres ENUM.
- PK strategy: TEXT canonical IDs.
- Read-model shape: canonical Scholardex-first.
- Default runtime remains Mongo-backed until H22.4.

## Migration Artifacts

Versioned migrations:
- `V1__h22_2_create_pg_enums.sql`
- `V2__h22_2_create_reporting_core_tables.sql`
- `V3__h22_2_create_reporting_core_indexes.sql`

Profile wiring:
- `application-postgres.properties` enables Flyway + PostgreSQL datasource in opt-in profile.

## ENUM Contract

Defined in `reporting_read` schema:
- `edition_normalized_enum`: `SCIE`, `SSCI`, `AHCI`, `ESCI`, `OTHER`, `UNKNOWN`
- `metric_type_enum`: `AIS`, `RIS`, `IF`
- `wos_source_type_enum`: `GOV_AIS_RIS`, `OFFICIAL_WOS_EXTRACT`

## Table Contract

### WoS read core

- `wos_ranking_view` (`journal_id` PK)
  - mirrors current `WosRankingView` read shape including normalized ISSN fields and build metadata.
- `wos_metric_fact` (`id` PK)
  - unique: `(journal_id, year, metric_type)`.
  - FK `journal_id -> wos_ranking_view.journal_id`.
- `wos_category_fact` (`id` PK)
  - unique: `(journal_id, year, category_name_canonical, edition_normalized, metric_type)`.
  - FK `journal_id -> wos_ranking_view.journal_id`.
- `wos_scoring_view` (`id` PK)
  - denormalized scoring lookup table aligned to `getTopRankings(...)` predicate.
  - unique: `(journal_id, year, category_name_canonical, edition_normalized, metric_type)`.
  - FK `journal_id -> wos_ranking_view.journal_id`.

### Scholardex reporting/admin read core

- `scholardex_publication_view` (`id` PK)
  - carries identity keys (`eid`, `wos_id`, `google_scholar_id`), text/date/search fields, array relationships, and build/link metadata.
- `scholardex_author_view` (`id` PK)
  - author display and affiliation arrays.
- `scholardex_forum_view` (`id` PK)
  - forum display and issue identifiers used in reporting/admin read paths.
- `scholardex_affiliation_view` (`id` PK)
  - affiliation/country lookup shape for reporting/admin surfaces.
- `scholardex_citation_fact` (`id` PK)
  - unique: `(cited_publication_id, citing_publication_id, source)`.
  - FKs to publication view for cited/citing IDs.
- `scholardex_authorship_fact` (`id` PK)
  - unique: `(publication_id, author_id, source)`.
  - FKs to publication and author views.
- `scholardex_author_affiliation_fact` (`id` PK)
  - unique: `(author_id, affiliation_id, source)`.
  - FKs to author and affiliation views.

## Index Contract

Required index coverage for existing query shapes:

- ISSN matching:
  - `wos_ranking_view(issn_norm)`,
  - `wos_ranking_view(e_issn_norm)`,
  - GIN on `wos_ranking_view(alternative_issns_norm)`.
- WoS scoring predicates:
  - `(metric_type, year, quarter, category_name_canonical, edition_normalized, journal_id)`,
  - `(journal_id, metric_type, year, edition_normalized)`.
- Publication identity lookup:
  - partial unique indexes for `eid`, `wos_id`, `google_scholar_id`.
- Reporting traversal:
  - citation indexes on `cited_publication_id`, `citing_publication_id`,
  - authorship indexes on `author_id`, `publication_id`,
  - author-affiliation indexes on `author_id`, `affiliation_id`.
- Search support:
  - lower(name/title)-based indexes,
  - GIN indexes on array relationship fields used in containment reads (`author_ids`, `affiliation_ids`).

## Constraint Contract

- Not-null on identity and edge join keys used by runtime reads.
- Unique constraints mirror existing Mongo uniqueness contracts on metric/category/edge facts.
- FK constraints are enabled for stable, non-circular relationships among read-core tables.

## Query Compatibility Mapping

Current Mongo read contracts covered by this schema:
- WoS ranking by normalized ISSN/eISSN/alternative ISSNs.
- WoS scoring distinct journal lookup for top-rankings by `(metricType, year, quarter, category, edition)`.
- Publication lookup by any identity key (`id`, `eid`, `wosId`, `googleScholarId`).
- Citation-by-cited publication traversal.
- Publication-by-author and publication-by-affiliation traversals via edge tables.
- Case-insensitive name/title lookups for reporting/admin read paths.
